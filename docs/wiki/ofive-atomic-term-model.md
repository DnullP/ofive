# ofive 原子词条模型

原子词条是 wiki 中最小的长期概念单元。它只解释一个概念、一个组件、一种状态流或一种治理对象，不同时承担多个主题。

原子词条的目标不是把文档切碎，而是让结构可解释：读者可以从总览页进入主题，再沿 wikilink 下钻到可复用的概念节点。

## 判断标准

一个词条可以独立成原子词条，通常满足以下条件：

1. 它在多个页面中被引用。
2. 它有稳定边界，不只是某个临时实现细节。
3. 它有明确 owner、关系和维护风险。
4. 它可以回答“是什么、不是什​​么、与谁相连、维护时注意什么”。

## 页面职责

- 总览页负责解释主题结构。
- 词汇表负责提供入口和简短定义。
- 原子词条负责解释边界、关系和治理要点。
- 专题架构页负责组织多个原子词条之间的协作模型。

## 写作规则

1. 标题对应一个概念，不混合多个概念。
2. 正文使用 wikilink 表达依赖关系。
3. 原子词条不写实现路径、命令或测试路径。
4. 如果需要说明实现定位，只在普通开发文档中处理。
5. 原子词条应优先复用已有词条，不重复定义同一概念。

## 结构关系

```text
项目总览
  -> 主题页
  -> 词汇表
  -> 原子词条
  -> 相关原子词条
```

这个结构让维护者可以从任意入口回到概念边界，也可以从某个概念继续追踪到它依赖的运行时、模块或治理对象。
